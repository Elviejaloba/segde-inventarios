# Security Policy

## Supported branch

- `main` is the only production-supported branch.

## Reporting a vulnerability

If you detect a security issue:

1. Do not publish it in a public issue.
2. Report details privately to repository maintainers.
3. Include reproduction steps, impact, and suggested mitigation if available.

## Secret handling

1. Do not commit credentials or API keys.
2. Rotate leaked credentials immediately.
3. Store secrets only in environment settings (GitHub/Railway/local secure store).
